# 레일CCTV 하드웨어/미디어 개발 계획

## 1. 프로젝트 목표

이 프로젝트는 CCTV 하드웨어 제어와 미디어 서버 구성을 담당한다.

최종 목표는 다음과 같다.

- 카메라 장치 인식
- RTSP 영상 송출
- 외부 클라이언트에서 영상 확인
- Pan/Tilt 등 CCTV 하드웨어 제어
- 자동 실행과 로그 확인 체계 정리

## 2. 개발 우선순위

```txt
카메라 인식
→ RTSP 송출
→ VLC 확인
→ 하드웨어 제어 신호 테스트
→ PTZ 동작 확인
→ systemd 자동 실행
→ 안정화
```

## 3. 프로젝트 구조

```txt
rail-cctv/
├── hardware-control/  # CCTV 하드웨어 제어
├── media-server/      # 카메라 입력과 RTSP 송출
├── deploy/            # systemd, 실행 스크립트, 운영 문서
├── docs/              # 설계 문서
└── README.md
```

## 4. 작업 단계

### 1단계: 범위 정리와 장비 확인

- 프로젝트 범위를 하드웨어 제어와 미디어 서버로 정리한다.
- 사용할 카메라 종류를 확인한다.
- 제어할 하드웨어 종류를 정리한다.
- 연결 장비와 운영 환경을 확인한다.

### 2단계: 카메라 영상 송출

- 카메라 장치 인식을 확인한다.
- FFmpeg 또는 GStreamer로 캡처를 테스트한다.
- RTSP 송출 방식을 결정한다.
- VLC에서 RTSP 영상을 확인한다.
- 성공한 명령어와 설정값을 문서화한다.

### 3단계: 미디어 서버 구성

- RTSP 포트와 스트림 경로를 정한다.
- 해상도, FPS, bitrate 기본값을 정한다.
- 서비스 재시작 시 스트림 복구 방식을 정리한다.
- 로그 확인 방법을 정리한다.

### 4단계: 하드웨어 제어

- 전원과 배선 구성을 확인한다.
- GPIO/PWM, 시리얼, 또는 장비 프로토콜 중 제어 방식을 정한다.
- Pan/Tilt 방향 제어를 테스트한다.
- 정지 명령과 안전 중단 조건을 정리한다.

### 5단계: 자동 실행과 안정화

- 미디어 서버 실행 스크립트를 작성한다.
- 하드웨어 제어 실행 스크립트를 작성한다.
- systemd 서비스를 등록한다.
- 재부팅 후 자동 실행을 확인한다.
- 장애 대응 절차를 문서화한다.

## 5. 1차 완료 기준

- 카메라가 장치에서 정상 인식된다.
- RTSP 스트림이 송출된다.
- VLC 또는 외부 클라이언트에서 영상을 볼 수 있다.
- 성공한 실행 명령어가 문서화되어 있다.

## 6. 최종 완료 기준

- 미디어 서버가 자동 실행된다.
- 하드웨어 제어 대상이 실제로 동작한다.
- 정지 명령 또는 안전 중단 방법이 확인된다.
- 포트, 스트림 경로, 로그 확인 방법이 문서화되어 있다.
